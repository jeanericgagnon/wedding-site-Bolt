Scoring rubric (1–5):
- **Clarity of value prop**: how fast a new user understands what they get.
- **CTA strength**: visibility + specificity + confidence of primary action.
- **Trust signals**: social proof, brand authority, breadth signals.
- **Template discoverability**: ease of browsing/finding a fitting template.
- **Customization depth cues**: visible proof of control/flexibility before sign-up.
- **Conversion friction**: lower is better flow; higher score = lower friction.

## Fast takeaways

1. **Category leaders win above-the-fold with “all-in-one” framing** (site + RSVP + registry + planning tools), not just “beautiful templates”.
2. **Template depth perception matters as much as actual depth**: even blocked template links still show rich “designs” entry points.
3. **Customization confidence is sold via concrete controls** (multi-event RSVP, private itineraries, widgets), not generic “customizable”.
4. **Trust is borrowed from adjacent value** (marketplace, vendors, stationery, registry scale), not only website product quality.
5. **Conversion friction drops when users can start at multiple entry points** (website, registry, guest list) from one landing surface.

## Implication for DayOf

- Keep builder UX simple/canvas-first, but top-level product framing should explicitly bundle:
  - website
  - RSVP/guest ops
  - registry integration
  - day-of command center
- On template entry surfaces, show depth quickly (families, styles, use-cases) before asking users to commit.
